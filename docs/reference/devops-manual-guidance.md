# DevOps Manual Guidance Snapshot

## Summary

DevOps Manual is the UDX source for operational controls, security, and delivery practices. Use it to align pipelines and policies with operational assurance.

## When To Use

- Defining delivery controls and gates.
- Auditing operational maturity.
- Establishing security and monitoring expectations.

## Quick Answers

- "How do we define delivery controls?" -> DevOps Manual.
- "What should we measure?" -> monitoring and observability guidance.
- "How do we treat supply chain risk?" -> DevOps Manual + supply chain ref.

## Focus Areas

- Security controls in delivery workflows.
- Supply chain security and dependency trust.
- Monitoring and observability practices.
- Automation for consistency and scale.
- Continuous compliance and SBOM practices.

## dev.kit Notes

- Use this to define default validation and reporting behaviors.
- Embed checks in workflows instead of manual steps.
